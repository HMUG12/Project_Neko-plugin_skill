# 国际化 (i18n) 指南

## 文件结构

```
i18n/
├── zh-CN.json    # 中文翻译
└── en.json       # 英文翻译
```

## 格式

两个 JSON 文件使用相同的 key 结构，值分别为对应语言的翻译。

```json
{
  "section.key": "翻译文本",
  "section.key_with_placeholder": "翻译文本 {param}"
}
```

## 命名规范

```json
{
  "panel.title": "设置面板标题",
  "panel.subtitle": "副标题",
  "panel.noAction": "动作不可用",

  "stats.fileCount": "已索引文件",
  "stats.totalSize": "索引总大小",

  "status.scanning": "扫描中…",
  "status.idle": "空闲",

  "form.scanConfig": "扫描配置",

  "fields.scanPaths": "扫描路径",
  "fields.scanPathsHelp": "帮助文本",

  "actions.updateSettings.label": "保存设置",
  "actions.triggerScan.label": "重新扫描",

  "messages.configSaved": "配置已保存。",
  "messages.scanStarted": "全量扫描已启动。",

  "error.operation_disabled": "文件操作未启用",
  "error.source_protected": "源路径在保护列表中: {path}",
  "error.permission_denied": "权限不足: {detail}"
}
```

命名模式：`类别.具体名称`，常见类别：
- `panel` — 面板级文本
- `stats` — 统计数据相关
- `status` — 状态文本
- `form` — 表单区块标题
- `fields` — 表单字段
- `actions` — 操作按钮标签
- `messages` — 成功/提示消息
- `error` — 错误消息

## 在 Python 中使用

```python
# 简单翻译
msg = self.i18n.t("messages.configSaved", default="配置已保存。")

# 带占位符
msg = self.i18n.t(
    "error.path_not_found",
    path=some_path,
    default=f"路径不存在: {some_path}"
)
```

**建议**：始终提供 `default` 参数作为后备，以防翻译文件缺失。

## 在 UI 中使用

```tsx
// 通过 props.t 使用翻译
const { t } = props
<Text>{t("panel.title")}</Text>
```

## 注意事项

1. **两个文件必须 key 完全一致** — 否则会出现未翻译的文本
2. **key 用点号分隔** — 不要用下划线或驼峰
3. **占位符用 `{name}` 格式** — 与 Python `str.format()` 兼容
4. **default 参数必须提供** — 防止翻译文件缺失时崩溃
5. **部署后同步** — 修改 i18n 文件后必须同步到部署目录

### Python 端安全调用

```python
# 包装 self.i18n.t() 防止翻译缺失时崩溃
def _t(self, key: str, default: str, **kwargs) -> str:
    try:
        return self.i18n.t(key, default=default, **kwargs)
    except Exception:
        return kwargs.get("default", default) if kwargs else default

# 使用
msg = self._t("error.source_protected", path=path,
              default=f"那个地方是禁区啦，不可以碰哦 🐾")
```
